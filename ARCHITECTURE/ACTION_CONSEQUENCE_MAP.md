# Action-Consequence Map

## Purpose

AEGIS earns trust by mapping what it is about to do, what it did, and what changed because of it.

## Required stages

1. Intent captured.
2. Planned action declared.
3. Expected consequence recorded.
4. Policy gate evaluated.
5. Execution performed.
6. Actual consequence recorded.
7. Outcome classified.
8. Memory candidate emitted.
9. Wealth impact updated.

## Trust-critical rule

Any action that can delete, spend, publish, expose secrets, change policy, or materially affect wealth requires a visible map entry and approval path.

## Minimum map fields

- actor
- action
- target
- expected outcome
- possible downside
- approval state
- actual outcome
- rollback option
- memory write decision

## Why it exists

Without this map, AEGIS becomes fast but untrustworthy.
With it, AEGIS can become powerful without becoming opaque.
