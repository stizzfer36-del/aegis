# Delegation Model 8-6-4-2-1

## Idea

The delegation model narrows work from broad possibility into a clean execution path.

- **8** — imagination breadth and option surface.
- **6** — system design and major route selection.
- **4** — subsystem decomposition.
- **2** — execution path selection.
- **1** — final action path.

## Goal

Reduce chaos without reducing ambition.

## Warden's role

Warden owns:

- delegation cleanliness,
- overlap detection,
- escalation order,
- veto when structure is violated,
- and prevention of agent sprawl.

## Rules

- Work must narrow as it approaches execution.
- Fewer actors are preferred near action.
- Agents may not spawn duplicate efforts.
- Any handoff must carry trace, memory, and acceptance criteria.

## Success condition

The user can inspect the narrowing path and understand why a task reached a specific executor.
