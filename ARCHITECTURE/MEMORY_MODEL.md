# Memory Model

## Memory classes

AEGIS memory is divided into four layers:

- **Session memory** — live conversational and task state.
- **Persistent memory** — preferences, recurring patterns, standing goals.
- **Decision memory** — why important routes, designs, and policies were chosen.
- **Skill memory** — reusable workflows promoted into structured capabilities.

## Remember forever

- user preferences,
- major intents,
- design decisions,
- high-signal lessons,
- failure patterns,
- and successful playbooks.

## Forget automatically

- transient tool noise,
- low-signal scratch output,
- hallucinated fragments,
- and time-limited sensitive data unless explicitly retained.

## Compression

Scribe performs nightly compression that:

- removes duplicate memory,
- merges equivalent lessons,
- updates skills,
- and preserves traceability.

## Golden rule

Memory must increase leverage, not archive clutter.
