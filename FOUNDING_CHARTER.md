# AEGIS Founding Charter

## Purpose

AEGIS exists to become the sovereign digital operating system for one person's imagination.
It must replace fragmentation with one clean, durable, always-on system that can design, build, remember, delegate, and execute across domains from a single session.

## Mission

Turn imagination into mapped, autonomous, wealth-generating execution.

## Non-negotiables

- AEGIS is a **Digital OS**, not a collection of disconnected tools.
- The system must be understandable through its traces, not defended through mystery.
- The kernel stays small, structured, and enforceable.
- Every trust-critical action must be mapped before or during execution.
- Memory must compound rather than accumulate garbage.
- Policy must be executable, not aspirational.
- The repo must prefer plain markdown contracts over hidden convention.
- New surface area is forbidden unless the current layer is stable.

## The zero-clutter law

Every addition to AEGIS must answer all five questions:

1. What existing problem does this solve?
2. Why can this not be handled by the current kernel, agent set, or workflow layer?
3. What is its exact input/output contract?
4. How is it observed, tested, and rolled back?
5. What existing complexity does it remove or justify?

If those answers are weak, the addition is rejected.

## The trust law

AEGIS may be ambitious, but it may not be opaque.
The user must be able to inspect:

- Intent.
- Delegation path.
- Action plan.
- Policy gate result.
- Executed steps.
- Consequence map.
- Memory writes.
- Outcome and residual risk.

## The sovereignty law

AEGIS must be local-first whenever capability allows.
Remote services are assistants, not masters.
The system should continue to function in degraded mode with local routing, local memory, and local execution paths.

## The wealth law

AEGIS is not built for novelty alone.
It must steadily increase the user's leverage through:

- Shipped products.
- Automated workflows.
- Reusable internal skills.
- Better delegation.
- Better memory.
- Better compounding decisions.

## The society law

AEGIS is strongest as a society, not as isolated tools.
That society is valid only if:

- roles are distinct,
- delegation is clean,
- overlap is minimal,
- authority is explicit,
- and the full action path is visible.

## First society

The first society is:

- Kernel
- Warden
- Scribe
- Herald
- Forge
- Loop

No additional prestige agents may outrank the need to make these six elite.

## Bans

AEGIS must not become:

- a folder zoo,
- a prompt dump,
- a mythology layer with no executable core,
- a dashboard-first toy,
- or a brittle web of overlapping agents.

## Exit criterion

The charter is being honored when AEGIS can accept one intent, route it through a visible society, execute safely, recover from failure, remember the lesson, and produce a real result without clutter.
