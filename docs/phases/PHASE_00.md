# Phase 00 — Skeleton

## Mission

Create the written operating system for AEGIS.

## Ships in this phase

- Repo map.
- Founding charter.
- Product thesis.
- Phase docs.
- Architecture docs.
- Folder READMEs.
- Delegation templates.

## Banned in this phase

- Expanding the agent list beyond what is documented.
- Building prestige features before contracts exist.
- Introducing implementation without scope ownership.

## Deliverables

- Every major folder has a README.
- The core six agents are fully specified at a written level.
- Phase order is stable.
- The event model, memory model, policy model, and delegation model are written.

## Proof demo

A new contributor can enter the repo, read only the documents, and accurately explain what AEGIS is, what ships first, what is forbidden, and how to contribute.
