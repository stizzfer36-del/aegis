# Phase System

AEGIS is built in controlled phases.
Each phase exists to make the next phase possible without clutter.

## Rules

- Only one active phase may shape the repo at a time.
- Later-phase ideas may be documented but not implemented ahead of proof.
- Every phase must define what is banned, not only what is desired.
- A phase is complete only when its proof demo is stable and repeatable.

## Phase order

- PHASE_00_SKELETON.md — document skeleton and operating rules.
- PHASE_01_KERNEL_CORE.md — event spine, mapping, policy, routing, scheduling, introspection.
- PHASE_02_UNIFIED_SESSION.md — Warden, Scribe, Herald, one session across surfaces.
- PHASE_03_EXECUTION_ENGINE.md — Forge + Loop and first real autonomous shipping loop.
- PHASE_04_REVENUE_ORGANS.md — revenue-producing workflows and economic instrumentation.
- PHASE_05_FULL_AGENT_SOCIETY.md — broader society expansion only after the core proves itself.
- PHASE_06_SELF_EXTENSION.md — controlled self-growth, skill promotion, new agents, self-authored improvements.

## Completion rule

A later phase cannot absorb the unresolved chaos of an earlier one.
